# kobun-tan 古文常識（背景知識）拡充

kobun-tan の読解 (layer 5) で「主語の把握」「心理変化」など読解戦略は伝えているが、**当時の文化・社会・宗教・風俗** といった本文に明示されない前提知識（古文常識）が欠落している。本スキルでこれを各教材へ補充する。

## 古文常識とは

**古文常識**＝古文を読解する上で前提となる、当時の文化・社会・風俗・宗教観など、本文には明示されていないが意味が成り立つために必要な背景知識。

### スコープ内（本スキルで扱う）

- **暦・時刻**: 時刻名（暁/つとめて/夜更く/有明）、月の異名、節句、年中行事
- **方角・空間**: 方違へ、寝殿造、御簾・几帳・障子の構造、京の方位観、歌枕
- **婚姻・家族**: 妻問い婚、三日夜の餅、後朝の文、元服・裳着、母方育ち
- **身分・官職**: 律令官職（蔵人・弁官・近衛・受領）、三位以上の貴族、女房の身分
- **宗教**: 仏教思想（末法思想、阿弥陀信仰、無常観）、出家・受戒、物忌み、神祇信仰、陰陽道
- **建築・調度**: 寝殿造、僧坊、御簾・几帳・屏風、調度品
- **装束**: 衣冠・束帯・直衣、十二単（女房装束）、童形、僧衣
- **食物・生活**: 当時の食物、夜食慣習、入浴慣習
- **物の怪・呪術**: 物忌み、加持祈祷、生霊・死霊、方違え
- **作品・ジャンルの特徴**: 説話集の傾向（仏教 vs 世俗滑稽）、軍記物の語り、日記文学の私性
- **登場人物の社会的位置**: 「児」「受領」「女房」など類型キャラクターの背景
- **作法・礼儀**: 起こし方、呼びかけ方、文の取り次ぎ方、上下の作法

### スコープ外（別スキルで扱う）

- **語彙の意味**: 重要古語の意味・古今異義 → `kobun-token-hints` で `token.hint` に
- **文法**: 助動詞・助詞・敬語の識別 → `kobun-token-hints` で `token.hint` に
- **本文の心理・主語**: 既存の layer 5 read points で扱う

## 参考実装

`public/texts-v3/chigo-no-sorane.json` — layer 5 points に `[古文常識]` プレフィックス付きで 6 項目を追加（児/比叡山と僧坊/かいもちひ/起こす作法/宇治拾遺物語のジャンル/体面意識）。MD `30-教材/古文/説話/ちごのそらね.md` の `### 古文常識（背景知識）` 節（項目 27〜32）が対応。

## 対象

- テキスト JSON: `F:\A2A\apps-released\kobun-tan\public\texts-v3\{id}.json`
- ミラー JSON: `F:\A2A\apps-released\kobun-tan\dist\texts-v3\{id}.json`（公開ビルド）
- 参照 MD: `F:\A2A\NotebookLM\kokugo-vault\30-教材\古文\**\*.md`（title で match）
- 追加先: `learningPoints.byLayer[layer:5].points` の末尾にエントリ追加 + MD「学習ポイント」配下に `### 古文常識（背景知識）` 節を新設

## 何項目追加するか

- **基本 4〜8 項目** が目安。文化的背景が豊富な作品（源氏物語・大鏡・平家物語など）は 8〜10 項目まで可
- 短編・教訓説話など背景が薄いものは 3〜4 項目で十分

## 抽出の判定基準

各教材について、以下の問いを立てる:

1. **本文に登場する固有名詞・場所名・建物名は何の象徴か？**（例: 比叡山＝天台宗根本道場）
2. **本文の登場人物の社会的位置は明示されているか？読者の前提知識を要するか？**（例: 児＝寺院に預けられた貴族の子）
3. **本文の出来事の理解に、当時の慣習・作法の知識が必要か？**（例: 起こす作法、文のやりとり）
4. **登場する物・食物・装束は当時どう位置づけられていたか？**（例: かいもちひ＝寺院の贅沢な夜食）
5. **作品ジャンル自体に読解上の特徴があるか？**（例: 宇治拾遺物語＝滑稽譚多し）
6. **本文の心理・行動を支える価値観は何か？**（例: 体面意識・末法思想・武士道）

## 追加形式

### JSON 側（layer 5 points 末尾に追記）

```json
{
  "text": "[古文常識] {見出し}: {2〜4 文の解説。150〜350 字目安}。{本作との関連を最後に明示}",
  "priority": "important"
}
```

- **必ず `[古文常識]` プレフィックス**を付け、既存の読解戦略 points と一見区別できるようにする
- 本文の文脈と無関係な辞書的解説に終わらず、最後に「本作との関連」を一文で結ぶ
- `priority` は `important` を基本（`essential` は読解戦略向け、`supplementary` は補助情報）

### MD 側（学習ポイント節配下に新設）

既存の `### 読解の要点` の直後、`## 1. 本文` の直前に以下を挿入:

```markdown
### 古文常識（背景知識）

本文には明示されていないが、読解に必要な当時の文化・社会・宗教の前提知識。

27. **{見出し}** ── {解説}

28. **{見出し}** ── {解説}

...
```

- 番号は **既存の読解の要点の続き番号**から開始（既存項目を破壊しない）
- 見出しは **太字**、解説は地の文で 150〜350 字
- MD `### 古文常識（背景知識）` 節は読解の要点の **下** に置き、本文より上に置く

## 実装手順

### 1. 対象教材の選定

未処理の hint カバレッジ ≥50% 教材から、文化的背景が読解に効くものを選ぶ。
特に効果的なのは:
- 寺院・宮廷を舞台にする作品（児・受領・女房・公達）
- 季節・年中行事が絡む作品
- 出家・婚姻・葬送など人生儀礼
- 軍記・説話・日記など作品ジャンル特有の文化前提

### 2. JSON を読む

```bash
node -e "const t=require('./public/texts-v3/{id}.json');console.log(JSON.stringify(t.learningPoints.byLayer.find(l=>l.layer===5),null,2))"
```

既存の layer 5 points と重複しないよう注意（特に「説話の教訓」「オチの構造」など本文の解釈レベルの points と区別）。

### 3. 対応 MD の確認

`F:\A2A\NotebookLM\kokugo-vault\30-教材\古文\**\{title}.md` の `### 読解の要点` セクションを確認。既に類似情報があれば省略。

### 4. 古文常識 4〜8 項目を執筆

判定基準の 6 問いに沿って項目を立て、各 150〜350 字で執筆。本作との関連を末尾で明示。

### 5. JSON Edit

`learningPoints.byLayer[layer:5].points` の末尾に `[古文常識]` プレフィックス付きで追加。
public/ と dist/ 両方を同期。JSON valid を保つ。

### 6. MD Edit

`### 古文常識（背景知識）` 節を新設し、項目番号は既存の読解の要点続きから。**既存項目は絶対に変更しない**。

### 7. 検証

```bash
node -e "JSON.parse(require('fs').readFileSync('public/texts-v3/{id}.json','utf8')); JSON.parse(require('fs').readFileSync('dist/texts-v3/{id}.json','utf8')); console.log('valid')"
```

### 8. コミット

```bash
git add public/texts-v3/{id}.json dist/texts-v3/{id}.json
git commit -m "feat: {教材名} 古文常識（背景知識）追加 + MD 逆反映"
git push origin main
```

MD は git 管理外（kokugo-vault は別 vault）のため git に乗せない。

## 並列化のコツ

- 1 教材につき agent 1 体、Edit ツールで JSON と MD を順次編集
- 並列 5 本程度が安定（rate limit 回避）
- スキル `kobun-token-hints` と並走させない（同一ファイルを編集する可能性）

## 注意事項

- **既存の layer 5 points を変更しない**。末尾に追加のみ
- **既存の MD 番号体系を破壊しない**。続き番号で追加のみ
- 解説は **辞書的定義に終わらず本作との関連を必ず示す**
- 重複する内容（既存の overview や読解の要点と被るもの）は省く
- 史実の正確性に注意。不確かな記述は避け、教科書レベルの定説に準拠
- 偏った主観や現代的価値判断（「当時は男尊女卑だった」等）は避ける

## 既存実装からの違い

`kobun-token-hints` がトークン単位の文法・語彙の点的ポイントを扱うのに対し、本スキルは **教材全体に通底する文化的前提** を面的に整理する。token.hint と本スキルは補完関係にあり、両方そろって初めて読解教材として完備する。
