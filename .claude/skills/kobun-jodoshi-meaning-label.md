# kobun-tan 助動詞 意味ラベル是正

読解教材（texts-v3 等）の助動詞の **意味ラベル**（過去/詠嘆、現在推量/原因推量 等）の取り違えを直す。
ポイントは2つ：**(A) 文法判別を正しく行う／(B) ラベルだけでなく "説明文" も含め全データ層を漏れなく直す**。

## A. 判別ルール

### A-1. けり ＝ 過去 / 詠嘆
- 原則：**地の文＝過去（伝聞過去）／和歌・会話文の文末＝詠嘆**。
- **詞書（前書きの散文）**の「申し**ける**時に」「降り**ける**を見て」等の連体形けるは**過去で正しい**。
  和歌集教材で過去・体のけるが大量にあっても大半は詞書。**疑うのは歌の結句（終・（結））のラベルだけ**。
- ✗ 和歌の結句（…なり**けり**／ぞ…**ける**／こそ…**けれ**）が「過去」 → ○ **詠嘆**に是正。
  - 実例：東下り「うつつにも夢にも人に逢はぬなり**けり**」過去→詠嘆（commit c54e6da）。
- **例外：地の文でも「発見・気づきの詠嘆」は詠嘆**（過去にしてはいけない）。
  - 「〜と見れば／思へば／問はすれば…なり**けり**」＝発見の詠嘆（袴垂「保昌といふ人なりけり」、若紫「尼なりけり」、方丈記「割り砕けるなりけり」等）。
  - 心内語・会話文中のけり、こそ…けれの気づき（絵仏師「かうこそ燃えけれと心得つる」）も詠嘆。
- **日記・物語の「語りの詠嘆体」**（源氏須磨「なりけり/にけり」連発、紫式部日記の情景末尾）は詠嘆で**維持**（ユーザー判断＝現状維持。明白な過去の誤ラベルではない）。
- 同じ段でも区別：東下り「見し人なり**けり**」は地の文の認識（…を見れば見知った人だった）＝**過去のまま**。

### A-2. らむ ＝ 現在推量 / 原因推量
- **現在推量**＝目に見えない現在の事態の推量「（今頃）〜しているだろう」。
- **原因推量**＝**眼前の事態**の原因・理由を、**疑問語（など・なぞ・いかに・いかで・何・いつとて 等）＋係助詞か/や**とともに推量「どういうわけで〜なのだろう」。
  - ✗ 「いつとて**か**…雪の降る**らむ**」が現在推量 → ○ **原因推量**。
    直前の地の文「雪いと白う降れり」で**雪が眼前にある**＋「いつとて」で理由を問う（commit 529f790）。
- **疑問語があっても**、不在の事態の場所・方向・行方を問う（「いづちゆく**らむ**」＝どこへ行くのだろう）のは**現在推量**。眼前事態の"理由"を問うときだけ原因推量。
- 疑問語なしの心内推量（児のそら寝「定めて…おどろかさむず**らむ**」＝きっと起こそうとするだろう）は**現在推量で正しい**。
- 「原因推量」は本教材の既存正式ラベル（けむ等でも使用）。用語はこれで統一。

## B. 全データ層の一括修正手順（最重要）

ラベルを1箇所直すだけでは不整合になる。**同じ語の説明が複数層に分散**しているので全部直す。

1. **texts-v3**（`public/` & `dist/`、`<id>.json`）
   - 当該 token の `grammarTag.meaning`（表示ラベル）
   - 当該 token の `hint`（解説文。meaningと食い違っていることが是正の手がかり）
   - **隣接 token の hint**（例：らむを直したら直前「降る」のhint「…の現在推量」も直す）
   - `learningPoints.byLayer[].points[].text` と `learningPoints.overview`（学習ポイントに同趣旨の記述）
2. **reading**（`public/` & `dist/`、`<id>.json`）：`annotations[].hints[].points[]`・`guide` の説明文
3. **guides**（`public/` & `dist/`、`<id>.json`）：`sections` 内の学習ポイント連結文（texts-v3 byLayer と同文のことが多い）
4. **vault MD**（`NotebookLM/kokugo-vault/30-教材/**.md`、**git管理外**）：品詞分解表 `| 語 | [[…\|ラベル]] |` ＋ 学習ポイント(文法の要点) ＋ **設問解答** ＋ 5-X重要語句解説 ＋ 現代語訳
5. `dist/` は **git非追跡のビルド成果物**。Vercelは `public/` からビルドするので **public が本番ソース**。両方直す（ローカル整合）。
6. **検証**：修正後に旧ラベルを全層 grep し、残っていないか確認。
   残ってよいのは「〜とは（現在推量とは）区別する」等の**対比説明だけ**。
7. **commit & push**（`public/` の変更、author=daichi223@gmail.com）。vault MD は別管理（コミット不要）。

詳しい層の構成：memory [[kobun-tan-kyozai-data-layers]]。

## C. 修正スクリプトの型

一回限りの是正は `scripts/_fix-<topic>.cjs` に置く（既存 `fix-<id>-*.cjs` 流儀）。
`['public','dist'] × ['texts-v3','reading','guides']` をループし、token特定で meaning/hint を書換え、
説明文は文字列 `split().join()` 置換。実例：`_fix-waka-keri.cjs`・`_fix-tojiramu-genin.cjs`・`_fix-tojiramu-setsumei.cjs`。

## D. 横断検出（他教材に同種の誤りがないか）

一時 `.mjs` で全 `public/texts-v3/*.json` を走査して候補抽出（読み取り専用、8GB機なので逐次・Workflow不使用）。
- 文(`originalText`)を **和歌/会話/地の文** に分類：和歌＝読点・鉤括弧・内部句点なしの16〜38字／会話＝「」を含む／他＝地の文。
- **けり詠嘆化候補**：和歌の文末（最終content token）で `meaning` に「過去」。
- **けり過去化候補**：地の文で `meaning`＝詠嘆（ただし発見の詠嘆・心内・会話・和歌は除外）。
- **らむ原因推量候補**：`meaning`＝現在推量 ＋ 文中に疑問語 ＋ 眼前の事態。
- 候補は機械判定で確定せず、文・訳・hintを見て一件ずつ判断（誤検出＝詞書・仮名序・設問選択肢が混じる）。

## 実績（2026-06-19, id=bfa5b23cf2 東下り宇津の山）
- けり：歌「逢はぬなりけり」過去→詠嘆（texts-v3/reading/MD、c54e6da）。全108教材走査で和歌文末けりの過去誤りは本件のみ。
- らむ：「雪の降るらむ」現在推量→原因推量（texts-v3/reading/guides/MD、529f790＋解説残し是正 bb0ac85）。
