### orphan_tail_only の修復方針

これらは sentences.originalText の総和が tokens の text 総和より少ない。考えられる対処:
1. **originalText を tokens から復元**: tokens を文単位で再分割して originalText を再生成
2. **末尾の余剰トークンを削除**: tokens のうち sentences の範囲外を捨てる（情報損失あり）
3. **手動で originalText を補完**: 教師用テキストや原典を確認して欠落部分を補う

例: `dd1c1e0d14 筑紫に、なにがしの押領使など` は originalText 30 字に対し tokens 254 字。本文がほぼ全部欠落して短い片言だけ残っている状態。→ originalText を全文復元するべき。

### orphan_tail+mismatch の修復方針

差分が小さい (1〜10 字) ものが大半。原文 (originalText) と tokens の連結を比較し、ズレ箇所を特定して pre-fix:
1. 全体の最長共通部分列 (LCS) を取って差分位置を抽出
2. tokens 側の text を必要なら修正 (typo / 句読点 / 仮名差) するか、新しい token を挿入して欠落を補う
3. 修正後に `fix-token-alignment.cjs` を再実行

差分が大きい (40 字超: `83e7b6a341 飽かぬ別れ` -61, `cbf3db0edb 能登殿の最期` -55, `3b84921e74 肝試し` -29 など) は手動精査推奨。

`b621d6ca53 土佐日記（門出）` は cover 269/269 = 0 差分なのに失敗。これは中間で局所的な挿入・削除があり全体長は一致しているケース。LCS による差分検出が必要。
