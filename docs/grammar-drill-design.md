# 文法道場 ドリル設計基準 v2

核思想: **「知識を言える」ではなく「文中で使える」を試す。**
perfect-math-lesson の基準（核の問い／メタ言い回し禁止／段階的例題／つまずきの対比）の古文版。

## 単元構成 — 各 8〜10 問・4段階（sort 順に固定）

| 段階 | 問数 | ねらい | kind |
|---|---|---|---|
| **S1 かたち** | 2 | 活用を具体形で使う。×「已然形はどれ」の表暗記 ○「『にけり』の『に』は何形？」 | katsuyo-fill |
| **S2 つなぐ** | 2 | 接続を**生成**で試す。「咲く＋完了ぬ→？」。誤答に混同形（咲かぬ）を置く | setsuzoku |
| **S3 よみとく** | 3〜4 | 例文で意味判別。**対比ペア**（咲きぬ／咲かぬ）を必ず作る | imi |
| **S4 まぎらわし** | 2〜3 | 同形他語との識別。判別手順を1行で言えるところまで | shikibetsu |

## 禁止事項

- **メタ問題単独**（「何形に接続する？」「〜の活用型は？」だけの問い）→ 接続・活用は生成問題（S1/S2）に変換する
- **語呂の選択問題** → 語呂は explanation で提示する側
- **選択肢に答えがそのまま入る**構成（「ザリ活用（ざら・ざり…）」等）
- **出典の捏造**。実在テキストの引用は public/grammar/*.json の textExamples 由来のみ。新作例文は「花咲きぬ」「年も暮れぬ」級の教科書的短文に限り、出典名を付けない

## 作問ルール

- 誤答（distractor）は**学習者が実際に混同する形**から取る（完了ぬ⇔打消ぬ⇔ナ変語尾、強意なむ⇔願望なむ⇔係助詞なむ…）
- explanation は**判別手順の再現**を1行で:「直前『咲き』＝連用形 → 完了」。正解の言い換えだけの解説は不可
- ref_heading はリファレンス JSON の section heading と完全一致（ジャンプ用）
- 対になる助動詞（ぬ⇔ず、つ⇔ぬ、き⇔けり）は両単元から相互に識別問題を出し、SRS 横断復習で混ぜられても解けるようにする

## 運用

- 投入: `node --env-file=.env.local scripts/apply-drills.mjs supabase/seeds/<batch>.json`（topic 単位で delete→insert、再実行可能）
- v2 適用順: ぬ・ず（サンプル済）→ 残り助動詞 → 助詞 → 係り結び → 敬語 → 識別 → 既存 v1（動詞・形容詞・む・けり・つ・べし）の作り直し
